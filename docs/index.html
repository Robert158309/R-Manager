R-Manager Documentation

What is R-Manager?

R-Manager is a desktop hybrid application, focused on managing electronic repair workshops. It includes modules for inventory control, repair tracking, and task management.

Overview

The application uses a web-based frontend (HTML, CSS, TypeScript), while C# manages logic and communication via WebView2.

Architecture

R-Manager follows a two-layer hybrid architecture:

  • C# Layer: Handles application logic, view loading, and system state management.
  • Frontend Layer: Provides the user interface using HTML, CSS, and TypeScript.

Communication between both layers is achieved through WebView2 using message-based events (postMessage).

System Flow

  1. User interacts with the UI (buttons, inputs, navigation)
  2. Frontend sends a message using postMessage via WebView2
  3. C# receives the message in the WebMessageReceived handler
  4. Backend processes the event based on the message type
  5. The UI is updated dynamically (e.g. view changes in #right container)

Design Decisions

The following design decisions were made to balance development speed, maintainability, and system flexibility:

  • WebView2 over native UI: Chosen to leverage web technologies for UI development while maintaining a desktop runtime environment.
  • Dynamic view loading: HTML views are loaded at runtime to enable modularity and reduce the need for recompilation when UI changes.
  • TypeScript integration: Introduced to improve frontend code quality through static typing and early error detection.

Notes

The frontend is tightly coupled to the C# runtime and operates exclusively through WebView2. It is not designed for standalone deployment and depends on the host application for execution, state management, and communication.

CODE SECTION

WebView2 Conection

This module starts de program execution, the minimum size of the window are 1024px x 590px, because the design can become distorted if it is reduced further. We also give the order that the window should always start maximized.

  
                public Form1()
                {
                    InitializeComponent();

                    this.MinimumSize = new Size(1024, 590);

                    this.Load += Form1_Load;

                    this.WindowState = FormWindowState.Maximized;

                }

Form Load

This module runs when the form loads and initializes WebView2 asynchronously to ensure the browser engine is ready before loading any content. It can optionally apply security settings such as disabling developer tools, context menus, and shortcut keys to limit user interaction with internal features. Then, it sets up an event to allow communication between the frontend in JavaScript and the backend in C#, enabling data exchange. After that, it dynamically builds the path to the main HTML file, which is the login page, and loads it into the WebView. Finally, it applies additional configuration to the browser environment. 

 
              private async void Form1_Load(object sender, EventArgs e)
              {
                  await webView21.EnsureCoreWebView2Async();

                  /*
                  // SECURITY / DEVTOOLS LIMITS 
                  webView21.CoreWebView2.Settings.AreDevToolsEnabled = false;
                  webView21.CoreWebView2.Settings.AreDefaultContextMenusEnabled = false;
                  webView21.CoreWebView2.Settings.IsStatusBarEnabled = false;
                  webView21.CoreWebView2.Settings.AreBrowserAcceleratorKeysEnabled = false;
                  */

                  webView21.CoreWebView2.WebMessageReceived += CoreWebView2_WebMessageReceived;

                  var basePath = Directory.GetParent(Application.StartupPath)
                      .Parent.Parent.Parent.FullName;

                  var path = Path.Combine(basePath, "frontend", "login.html");

                  webView21.CoreWebView2.Navigate(new Uri(path).AbsoluteUri);

                  ConfigWebView();
              }
